# 要件定義書

## はじめに

このスペックは、Biomeリンターの`lint/suspicious/noThenProperty`エラーを修正し、Supabaseモックファイルをより適切な実装に変更することを目的としています。現在のモック実装では`then`プロパティを直接オブジェクトに追加しており、これがPromiseライクなオブジェクトとして誤解される可能性があります。

## 要件

### 要件1

**ユーザーストーリー:** 開発者として、Biomeリンターエラーが発生しないクリーンなコードベースを維持したい。そうすることで、コードの品質を保ち、CI/CDパイプラインが正常に動作する。

#### 受け入れ基準

1. WHEN Biomeリンターを実行した時 THEN `lint/suspicious/noThenProperty`エラーが発生しない
2. WHEN `__tests__/mocks/supabase.ts`ファイルをチェックした時 THEN すべてのリンターエラーが解決されている
3. WHEN pre-commitフックが実行された時 THEN リンターエラーによる失敗が発生しない

### 要件2

**ユーザーストーリー:** 開発者として、Supabaseクライアントの動作を正確にモックしたい。そうすることで、テストが実際のSupabaseクライアントの動作を適切に反映する。

#### 受け入れ基準

1. WHEN データベース操作をモックした時 THEN 実際のSupabaseクライアントと同様のPromiseベースのAPIを提供する
2. WHEN チェーンメソッド（select, eq, insert等）を使用した時 THEN 適切なPromiseオブジェクトが返される
3. WHEN 非同期操作をテストした時 THEN await/async パターンが正常に動作する

### 要件3

**ユーザーストーリー:** 開発者として、既存のテストコードに影響を与えずにモック実装を改善したい。そうすることで、テストの安定性を保ちながらコード品質を向上させる。

#### 受け入れ基準

1. WHEN モック実装を変更した時 THEN 既存のテストが引き続き正常に動作する
2. WHEN テストを実行した時 THEN すべてのテストケースがパスする
3. WHEN モックのAPIを使用した時 THEN 以前と同じインターフェースが提供される
