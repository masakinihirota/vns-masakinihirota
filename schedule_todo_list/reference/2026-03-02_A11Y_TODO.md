## 🎯 5つの最高目標（WCAG 2.1 AA準拠）

1. **知覚可能性（Perceivable）**: 色・コントラスト・テキストサイズが視認可能（高齢者対応）
2. **操作可能性（Operable）**: キーボードのみで全操作可能 + Tab順序が任意で定義可能
3. **理解可能性（Understandable）**: エラー・メッセージが明確 + セマンティクスが正しい
4. **堅牢性（Robust）**: スクリーンリーダーが完全に理解可能 + ARIA属性が正確
5. **継続性（Sustainable）**: vitest-axeで自動監視 + 定期監査基準（月1回）で維持

```typescript
// src/__tests__/accessibility/templates/component-a11y-template.ts
import { render } from '@testing-library/react';
import { axe, toHaveNoViolations } from 'vitest-axe';
import { expect, describe, it } from 'vitest';

expect.extend(toHaveNoViolations);

/**
 * コンポーネントアクセシビリティテストテンプレート
 * 各コンポーネントテストは以下を参考に実装
 */
export function testComponentAccessibility<T extends React.ComponentType<any>>(
  ComponentUnderTest: T,
  componentName: string,
  testCases: Array<{ name: string; props: React.ComponentProps<T> }>
) {
  describe(`${componentName} Accessibility`, () => {
    testCases.forEach(({ name, props }) => {
      it(`should have no violations - ${name}`, async () => {
        const { container } = render(<ComponentUnderTest {...props} />);
        const results = await axe(container);
        expect(results).toHaveNoViolations();
      });
    });
  });
}
```

**使用例**:
```typescript
// src/__tests__/accessibility/button.test.ts
import { testComponentAccessibility } from './templates/component-a11y-template';
import { Button } from '@/components/ui/button';

  { name: 'default', props: { children: 'Click me' } },
  { name: 'disabled', props: { children: 'Click me', disabled: true } },
  { name: 'with aria-label', props: { 'aria-label': 'Close dialog', children: 'X' } },
]);
```

```tsx
function CustomTabs() {
  const [activeTab, setActiveTab] = useState(0);
  const tabs = ['Tab1', 'Tab2', 'Tab3'];

  const handleKeyDown = (e: React.KeyboardEvent) => {
    let nextTab = activeTab;
    if (e.key === 'ArrowRight') {
      nextTab = (activeTab + 1) % tabs.length;
    } else if (e.key === 'ArrowLeft') {
      nextTab = (activeTab - 1 + tabs.length) % tabs.length;
    }
    if (nextTab !== activeTab) {
      setActiveTab(nextTab);
    }
  };

  return (
    <div onKeyDown={handleKeyDown}>
      {tabs.map((tab, i) => (
        <button
          key={i}
          onClick={() => setActiveTab(i)}
          aria-selected={i === activeTab}
          role="tab"
        >
          {tab}
        </button>
      ))}
    </div>
  );
}
```

### 8. アニメーション / 動き（`prefers-reduced-motion`）

#### 8.1 `prefers-reduced-motion` 尊重
**目的**: 前庭障害・モーション感応性発作ユーザーへの配慮

```css
{/* ❌ 悪い（常にアニメーション） */}
.button {
  transition: all 0.3s ease;
}

{/* ✅ 良い（ユーザー設定尊重） */}
@media (prefers-reduced-motion: no-preference) {
  .button {
    transition: all 0.3s ease;
  }
}

@media (prefers-reduced-motion: reduce) {
  .button {
    transition: none;  /* またはすぐに変化 */
  }
}
```

**React例**:
```tsx
function useMotion() {
  const [isReduced, setIsReduced] = useState(false);

  useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setIsReduced(mediaQuery.matches);

    const handleChange = (e: MediaQueryListEvent) => setIsReduced(e.matches);
    mediaQuery.addEventListener('change', handleChange);
    return () => mediaQuery.removeEventListener('change', handleChange);
  }, []);

  return isReduced;
}

{/* 使用例 */}
const isMotionReduced = useMotion();
<button
  className={isMotionReduced ? '' : 'animate-pulse'}
>
  Save
</button>
```

**チェックリスト**:
- [ ] すべてのアニメーション（transition, animation, fade等）に `prefers-reduced-motion` 适用
- [ ] アニメーション有効時は smooth （0.3-0.5s）
- [ ] アニメーション無効時は instant（不可視にするか 0s に）
- [ ] macOS/Windows の高コントラストモード設定で確認

**見積もり**: 2-3時間

**テンプレート**:

```typescript
// src/lib/error-messages.ts
export const errorMessages = {
  INVALID_EMAIL: 'メールアドレスが正しくありません。例: user@example.com',
  PASSWORD_TOO_SHORT: 'パスワードは8文字以上で設定してください',
  PASSWORD_WEAK: 'パスワードは大文字・小文字・数字を含む必要があります',
  REQUIRED_FIELD: 'このフィールドは必須です',
  NETWORK_ERROR: 'ネットワークエラーです。接続を確認してから再試行してください',
  SERVER_ERROR: 'サーバーエラーです。しばらく待ってから再度試してください',
};

// 使用例
function LoginForm() {

  const validateEmail = (email: string) => {
    if (!isValidEmail(email)) {
      setErrors(prev => ({
        ...prev,
        email: errorMessages.INVALID_EMAIL
      }));
    }
  };

  return (
    <form>
      <div>
        <label htmlFor="email">メールアドレス</label>
        <input id="email" type="email" onChange={(e) => validateEmail(e.target.value)} />
        {errors.email && (
          <span id="email-error" role="alert">
            {errors.email}
          </span>
        )}
      </div>
    </form>
  );
}
```

- [ ] 記号・絵文字の乱用を避ける（説明的な文

## ⚠️ よくある落とし穴・注意点

### 1. "aria-hidden で解決"という思考停止
❌ **悪い例**:
```tsx
<button>
  <SaveIcon aria-hidden="true" />  {/* アイコンを隠した */}
  保存
</button>
```

✅ **正しい修正**:
```tsx
<button aria-label="設定を保存して続行">
  <SaveIcon />
  保存
</button>
```

法則: **aria-hidden は装飾要素（divider, spinner等）のみ。コンテンツアイコンは隠さない**

---

### 2. Radix UI のデフォルト ARIA が複製される
❌ **悪い例**:
```tsx
<Button role="button" aria-label="Save">Save</Button>
```

✅ **正しい**:
```tsx
<Button aria-label="Save">Save</Button>
<Button>Save</Button>  {/* テキストで十分ならaria-label不要 */}
```

法則: **Radix UI は既にARIA対応。追加は最小限に**

---

### 3. `tabIndex > 0` で Tab 順序を制御する
❌ **悪い例**:
```tsx
<button tabIndex={5}>Save</button>
<button tabIndex={3}>Cancel</button>
```

✅ **正しい**:
```tsx
<button>Save</button>
<button>Cancel</button>
{/* DOM順序 = Tab順序。表示順序変更は CSS Flexbox */}
```

法則: **tabIndex ≤ 0 のみ。順序制御は DOM / CSS に委ねる**

---

### 4. Escape キーをすべてのボタンに実装
❌ **悪い例**:
```tsx
<button onKeyDown={(e) => e.key === 'Escape' && handleClose()}>
  一般的なボタン
</button>
```

✅ **正しい**:
```tsx
{/* モーダル・ドロップダウン内のボタンのみに Escape */}
<Dialog onEscapeKeyDown={handleClose}>
  <button>Close</button>
</Dialog>
```

法則: **Escape は「閉じる」「キャンセル」シーンのみ**

---

### 5. prefers-reduced-motion を無視する
❌ **悪い例**:
```css
.menu { animation: slide 0.3s ease; }
```

✅ **正しい**:
```css
@media (prefers-reduced-motion: no-preference) {
  .menu { animation: slide 0.3s ease; }
}
```

法則: **すべてのアニメーションに prefers-reduced-motion 対応**

---

### 6. 色対比を複数テーマで検証しない
❌ **悪い例**: ライトテーマで合格、ダークテーマで NG

✅ **正しい**:
```typescript
// tailwind.config.ts でテーマ毎に色を定義
{
  light: { text: '#1f2937' },      /* ライト用 */
  dark: { text: '#f3f4f6' },       /* ダーク用 */
}
```

法則: **各テーマで WebAIM checker 実行**

---

### 7. フォーカストラップを考慮しない
❌ **悪い例**:
```tsx
<Dialog>
  <input />
  {/* Tab でダイアログ外に脱出可能 */}
</Dialog>
```

✅ **正しい** (shadcn/ui Dialog は自動対応):
```tsx
<Dialog>{/* Radix Dialogが自動的に focus trap実装 */}</Dialog>
```

法則: **Modal / Dialog は focus-trap + focus-restore が必須**

---

### 8. エラーメッセージに aria-live がない
❌ **悪い例**:
```tsx
{errorMessage && <span>{errorMessage}</span>}
```

✅ **正しい**:
```tsx
<div aria-live="assertive" role="alert">
  {errorMessage && <span>{errorMessage}</span>}
</div>
```

法則: **動的コンテンツ更新には aria-live**
