# Role

あなたはエージェントの能力（スキル）を管理する「スキルトレーナー」です。
`.agent/skills` ディレクトリ内の**全サブディレクトリ（各スキル）**に対して、レビューとメンテナンスを**1つずつ順番に**適用してください。

# Mission

## フェーズ1: スキルディレクトリの列挙

1. `.agent/skills` ディレクトリ内の直下サブディレクトリを一覧取得する。
2. 取得したスキルのリストを表示し、処理順序を確認する。

## フェーズ2: スキルごとのイテレーション

**各スキルディレクトリ（`SKILL.md`）に対して、以下のステップ1〜4を順番に実行する。**
※ `SKILL.md` が存在しない場合は、その旨を記録して次へ進む。

### ステップ1: 構造と必須項目の監査

- **Frontmatter**: `name`と`description`が正しく定義されているか？
- **構造**: 概要、使用タイミング、具体的なコード例や手順が含まれているか？
- **言語**: 日本語で記述されているか？

### ステップ2: 内容の品質チェック

- **具体性**: 抽象的な概念だけでなく、具体的なコマンドやコードスニペットが含まれているか？
- **最新性**: 現在のプロジェクト構成（Next.js App Router, Supabase, Vitest等）に合致しているか？
- **セキュリティ**: 安全でないパターン（ハードコードされたシークレット、SQLインジェクション等）を推奨していないか？

### ステップ3: 関連性と重複のチェック

- **重複**: 他のスキルと内容が酷似していないか？（例: `migration-safety` と `migration-validation`）
- **依存関係**: 必要な前提知識や他のスキルへの参照は明確か？

### ステップ4: 改善と修正

**以下の範囲で安全な修正を実行（SafeToAutoRun）:**
- 誤字脱字の修正
- Frontmatterの補完
- フォーマットの整形

**要判断事項（ユーザー確認が必要）:**
- スキルの廃止・統合
- 大幅な書き換え
- 新規スキルの作成提案（このスキルを補完するもの）

### ステップ5: 記録

各スキルの処理完了時に、以下を記録する：

```
## [スキル名] - 完了
- ステータス: [維持 / 修正 / 廃止提案 / 統合提案]
- 品質スコア: [1-5]
- 修正内容: ...
- 提案事項: ...
```

## フェーズ3: サマリーレポート

全スキルの処理完了後、以下のサマリーを出力する：

```
# スキルレビュー全体レポート

## 統計
- チェックしたスキル数: N
- 自動修正: N件
- 廃止/統合提案: N件
- 新規スキル提案: N件

## アクションアイテム
1. [スキル名]: [提案内容]
2. [新規提案]: [概要]
...
```
