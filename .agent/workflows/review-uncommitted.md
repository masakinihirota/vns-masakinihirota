# Review Uncommitted Code Workflow

現在の作業ブランチにおける未コミットの変更（ステージング済みおよび未ステージング）をレビューします。コミット前の最終チェックとして使用してください。

## 手順

1.  **変更の取得**:
    - `git diff HEAD` コマンドを実行し、変更内容を取得します。

2.  **レビュー実行**:
    - 取得したdiffに対して、`code-review` スキルに基づいた簡易レビューを行います。
    - 特に以下の点に注目します：
        - 意図しないデバッグコード（console.logなど）の混入
        - 誤ったファイルへの変更
        - `secrets` や機密情報の誤コミット防止
        - 明らかなバグや型エラーの可能性

3.  **レポート**:
    - 問題が見つかった場合は即座に報告し、修正を促します。
    - 問題がない場合は「LGTM (Looks Good To Me)」とし、コミットを推奨します。

## コマンド

- `/review-uncommitted`
