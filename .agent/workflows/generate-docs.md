## 実行手順

このワークフローは、Antigravity（AIエージェント）が「実際のアプリ実装（コード/UI）」と「設計（要件定義書）」を比較し、利用者が「現時点で何ができ、何が将来の予定か」を把握するための利用者向けドキュメントを `/doc` ディレクトリ内に作成するためのものです。

1. **実装済み機能の特定（実態調査）**:
   - `src/app` および `src/components` ディレクトリを走査し、実際に定義されているルート（画面）とUIコンポーネントを特定します。
   - 画面上のテキスト、ボタン、ナビゲーションから「現在ユーザーが操作可能な機能」を詳細に抽出します。

2. **未実装機能の特定（将来計画の抽出）**:
   - `anti-design` ワークツリー内の `0001-01-全体要件定義書.md` および `0001-02-全体要件設計書.md` を解析し、定義されている全機能をリストアップします。
   - ステップ1で見つからなかった機能を「将来実装予定の機能」として抽出します。

3. **実装ステータスの分類**:
   - 抽出した全機能を以下のステータスで分類し、利用者が一目で進捗を理解できるようにします。
     - `[実装済み]`: 画面があり、実際に機能が動作するもの。
     - `[未実装/計画中]`: 要件定義には存在するが、コード上に実体がないもの。

4. **利用者向けドキュメントの生成/更新**:
   - 以下の構成でMarkdownファイルを生成または更新します。
     - `doc/features.md`: 実装状況表を含む機能一覧。ステータスに基づき、アプリの「現在地」を説明。
     - `doc/user-guide.md`: 実装済みの機能のみを対象とした、現在の操作ガイド。
     - `doc/updates.md`: 最近のコミットや追加された画面に基づいた更新情報。

5. **平易な日本語への変換**:
   - 技術用語（例: エンティティ、RLS、RSC）は一切使わず、「名刺が作れる」「仲間を探せる」といった利用者のメリットに基づいた言葉で記述します。

// turbo 6. **生成の実行**:

- `write_to_file` ツールを使用して、解析結果を `/doc` ディレクトリに書き出します。

## コマンド

`/generate-docs` を入力することで、Antigravityがこのワークフローを即座に開始します。
