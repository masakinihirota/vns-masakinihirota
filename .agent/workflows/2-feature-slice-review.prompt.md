# Feature Slice Review Workflow

このワークフローは、特定のユースケース（機能）を指定し、UIの見た目からバックエンド（DB取得ロジック）まで、データの流れ（Data Flow）に沿って一貫した整合性があるかを垂直（Vertical Slice）にレビューします。

## 対象ディレクトリ・ファイル（推奨）

- 実行時にユーザーが指定する機能に関連する一連のファイル群。
- 例: `src/app/sample/` 関連ページ、`src/components/profile-list/` 関連コンポーネント、およびそれに依存する `lib` ファイル。

## 手順

1.  **機能スコープの定義**: ユーザーからのコマンド引数、または対話を通じて、対象となる機能の起点（Page）から終点（Database/Logic）までのファイル群を特定します。
2.  **機能の一貫性チェック**: 以下の観点で垂直レビューを行います。
    - **Data Flow Integrity**: データベースからUIまでのデータの受け渡しに不備（不要な型変換、オプション引数の処理漏れ、不適切な非同期処理）がないか。
    - **TDD Flow**: UIテスト（`.test.tsx`）とロジックテスト（`.logic.test.ts`）で役割分担ができているか。データフェッチを含む結合テストが存在し、機能要件を満たしているか。
    - **Error Handling & UX**: 例外発生時のキャッチ漏れがないか。ユーザーフレンドリーなエラーメッセージや、ローディング状態・バックグラウンド同期時の適切なフィードバックが実装されているか。
    - **YAGNI & Complexity**: その機能にとって過剰な実装（不要なインターフェース、深すぎるコンポーネントネスト）はないか。
3.  **全体最適化の提案**: 特定のレイヤーに不自然に責務が偏っている場合（例：UIコンポーネントの中に複雑なビジネスロジックがある）、アーキテクチャ原則（`directory-structure.md`）に沿ったリファクタリング案を提示します。

## コマンド例

- `/feature-slice-review プロフィール作成機能`: プロフィール作成に関連する一連の処理の流れをレビュー
- `/feature-slice-review src/app/login`: ログイン経路に関連する画面・ロジックを一貫してレビュー
