# UI/UX & Design Guidelines

ユーザーインターフェースのデザイン、テーマ、およびユーザーフィードバックに関するガイドラインです。

## 1. Design Theme (デザインテーマ)

デジタル庁UIの構造思想を取り入れつつ、視覚的には以下のテーマを適用します。

### Light Mode: Glassmorphism (グラスモーフィズム)

- **コンセプト**: 明るく柔らかいグラデーションと、ガラスのような半透明・ぼかし効果。
- **背景**: 明るいグラデーション（例: `from-white via-blue-50 to-purple-50`）。
- **カード/モーダル**: `bg-white/10 backdrop-blur-md border border-white/20 shadow-lg`。
- **アクセント**: 青/紫系（例: `#4F46E5` / `#7C3AED`）。
- **制約**: 可読性を最優先し、コントラスト比4.5:1以上を確保すること。

### Dark Mode: Elegant (エレガント)

- **コンセプト**: 上品で控えめな暗色、タイポグラフィ重視。
- **背景**: 深いニュートラル（例: `bg-[#0B0F1A]`）。
- **カード**: 上品な暗色ガラス（例: `bg-white/5 backdrop-blur-md`）。
- **テキスト**: `text-neutral-200` を基本とし、白(#FFF)の使用は避ける。
- **実装**: `.dark` クラスとCSS変数（トークン）を用いて実装・切替を行う。

## 2. Implementation Strategy (実装戦略)

- **CSS変数**: テーマ用トークン（`--bg`, `--card`, `--text`, `--accent`）を定義し、ライト/ダークで値を切り替える。
- **Tailwind**: ユーティリティクラスを活用するが、色はトークン経由（例: `bg-[var(--bg)]` や `bg-background`）で指定する。
- **コンポーネント優先**: `Card`, `Button` などの共通コンポーネントから置換を進める。

## 3. UX Feedback & Error Messages (フィードバック設計)

高齢者やITに不慣れなユーザーに配慮し、優しくわかりやすいフィードバックを提供します。

### メッセージ記述ルール

- **専門用語禁止**: "Invalid Token", "500 Error" などは禁止。「ログイン有効期限が切れました」「サーバー側で一時的な問題が発生しました」のように平易な言葉に変換する。
- **解決策の提示**: 「失敗しました」だけでなく、「どうすればよいか」をセットで提示する。
- **トーン**: 丁寧語（「～です」「～ます」）で統一し、威圧感のある記号や色は避ける。

### 視覚的フィードバック

- **Loading**: 通信・処理中は必ずローディング（スピナーやスケルトン）を表示する。
- **トースト**: 操作完了時は必ず成功/失敗のフィードバックを表示する。
  - **成功**: 数秒で消える緑/青のトースト。
  - **失敗**: ユーザーが閉じるまで消えない赤/オレンジのメッセージ（読み飛ばし防止）。

### 実装パターン

- **フォームバリデーション**: `onBlur` や `onChange` でリアルタイムに、平易な日本語でエラーを表示する。
- **エラーハンドリング**: `try-catch` で捕捉したエラーを、ユーザーフレンドリーなメッセージに変換するユーティリティを使用する。
