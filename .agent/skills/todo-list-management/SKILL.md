# TODO List Management Skill

## 概要

TODOリストの生成、追跡、整理を体系的に行うためのスキル。プロジェクト内に散在するTODOを一元管理し、完了・進行中・未実施を明確に区分して、プロジェクト進捗の可視化を実現します。

---

## 🎯 目的

1. **TODOドキュメントの散在化を防止** - 各タスクが適切なファイルに格納される
2. **完了度の把握を容易に** - ✅ / ⏳ / ❌ で瞬時に判定可能
3. **アーカイブ戦略を統一** - 完了・レビュー済みは archive へ自動整理
4. **残タスク管理の効率化** - 途中のタスクから新リストを自動生成
5. **歴史的記録の保持** - 過去の意思決定プロセスを archive に保存

---

## 📋 実装ガイドライン

### Phase 1: TODOファイルの初期作成

**タイミング**: プロジェクト段階・スプリント開始時

**作成方法**:
```
schedule_todo_list/YYYY-MM-DD_PROJECT_NAME_TODO.md
```

**必須セクション**:
1. **Header**
   - 生成元ドキュメント（link）
   - 作成日時
   - 更新ルール

2. **優先度分類**
   - 🔴 高（必須、締切あり）
   - 🟡 中（重要、締切柔軟）
   - 🟢 低（おまけ、いつでも可）

3. **各TODO項目**
   ```markdown
   ### N. [タスク名] 🔴/🟡/🟢 ✅/⏳/❌

   目的: ...
   ステップ:
   - [ ] N.1: ...
   - [x] N.2: ...

   完了条件:
   - ✅ 条件1
   - ✅ 条件2
   ```

4. **進捗トラッキング** (定期更新)
   ```markdown
   完了タスク: N個 ✅
   進行中: M個 ⏳
   未実施: K個 ❌
   完了度: XX%
   ```

---

### Phase 2: 進行中のトラッキング

**毎日の更新**:
1. 実施開始時に `⏳` マークをチェック
2. 各ステップで `[x]` にマーク
3. ステップ完了時に進捗パーセンテージを更新

**例**:
```markdown
### 1. スキーマ統合 🔴 ⏳
進捗:
- [x] 1.1: スキーマ調査
- [x] 1.2: マイグレーション作成
- [ ] 1.3: FK更新
- [ ] 1.4: データマイグレーション
完了度: 50% (2/4)
```

---

### Phase 3: 完了度判定と分類

**判定基準**:

#### ✅ 完了（ARCHIVED）
条件:
- [ ] すべてのチェックボックスが `[x]`
- [ ] 完了条件をすべて満たしている
- [ ] テスト/ビルドが成功している

**アクション**:
- ファイル移動: `schedule_todo_list/` → `schedule_todo_list/archive/`

#### ⏳ 進行中（ACTIVE）
条件:
- [ ] 1個以上のステップが実施中
- [ ] 完了度が 1～99%

**アクション**:
- ファイル名に進捗を記載: `2026-03-01_BACKLOG.md` など
- 関連ドキュメントにリンク

#### ❌ 未実施（BACKLOG）
条件:
- [ ] すべてのチェックボックスが `[ ]` のままか
- [ ] 優先度が低い場合もある

**アクション**:
- 新規バックログファイルに集約: `2026-03-01_BACKLOG.md`

---

### Phase 4: アーカイブ戦略

**アーカイブ対象**:
1. ✅ **完了したTODO** - すべてのタスクが完了
2. 📋 **参考用ドキュメント** - レビュー結果、比較分析、修正オプションなど（直接実装に使用しない）
3. 🗑️ **廃止されたファイル** - 修正版が存在するなど、不要になった計画

**アーカイブ方法**:
```bash
# 単一ファイルの移動
mv schedule_todo_list/COMPLETED_FILE.md schedule_todo_list/archive/

# 複数ファイルの一括移動（PowerShell）
$files = @('FILE1.md', 'FILE2.md');
$files | ForEach-Object {
  Move-Item -Path "schedule_todo_list\$_" -Destination "schedule_todo_list\archive\"
}
```

**アーカイブREADME**:
- ファイル名、内容、完了日をリスト
- 読むべき順序を記載
- 参照方法を明記

---

### Phase 5: 新規バックログの生成

**タイミング**: 古いTODOを整理する時前後で **3～5日ごと**

**手順**:
1. 各TODOファイルの完了度を確認
2. 完了まで残りステップを抽出
3. 新バックログを生成（`YYYY-MM-DD_BACKLOG.md`）
4. 未実施タスクはそのまま残す

**新バックログの構成**:
```markdown
# YYYY-MM-DD - アクティブ バックログ

## 🔴 CRITICAL - 本番展開前
### 1. [タスク名]
- [ ] 1.1: ...
- [ ] 1.2: ...

## 🟠 MAJOR - リリース前
### 4. [タスク名]
- [x] 4.1: ... (完了)
- [ ] 4.2: ...

## 📊 進捗サマリー
| カテゴリ | 完了 | 進行中 | 未実施 |
|---------|------|--------|--------|
| CRITICAL | 2 | 0 | 1 |
```

## 🔍 スキル使用例

### 例1: Hono統合タスクの整理（2026-03-01実施）

**開始状態**:
- ❌ `2026-03-01_HONO_INTEGRATION_TODO.md` - レビュー指摘が多い
- ⏳ `schedule_todo_list/2026-03-01_HONO_MASTER_PLAN.md` - 修正版、進行中

**実施内容**:
```
1. 2026-03-01_HONO_INTEGRATION_TODO.md を review
   → レビュー指摘が多い = 参考用ファイル
   → archive/ へ移動

2. 関連レビューファイルも archive/
   - 2026-03-01_HONO_REVIEW_CRITIQUE.md
   - 2026-03-01_HONO_REVIEW_SUMMARY.md
   - 2026-03-01_HONO_MASTER_PLAN_FIX_OPTIONS.md
   - 2026-03-01_HONO_IMPLEMENTATION_REVISED.md

3. 2026-03-01_HONO_MASTER_PLAN.md は schedule_todo_list/ に残す
   → 現在のアクティブプラン

4. 新バックログ作成: 2026-03-01_BACKLOG.md
   → 残りCRITICAL/MAJOR/MINORタスク集約
```

**結果**:
- `schedule_todo_list/`: 5個のアクティブファイル
- `archive/`: 13個の完了・参考ドキュメント

---

### 例2: セキュリティ修復タスク（2026-02-28～03-01実施）

**開始状態**:
- ❌ `SECURITY_FIX_TODO.md` - 多くのタスク未実施

**実施内容**:
```
2026-02-28: 複数のセキュリティタスク実施 (CRITICAL + MAJOR)
2026-03-01: 残りのセキュリティタスク + テスト追加

すべてのチェックボックス ✅ → 完了
```

**アクション**:
```bash
# 完了したファイルをarchiveに移動
mv SECURITY_FIX_TODO.md archive/
```

**新バックログに反映**:
- ✅ SECURITY_FIX_TODO で実施した内容を新バックログから削除
- スキーマ修復の依存関係を確認

### Phase 6: セッション終了時プロトコル 🎯

**目的**: セッション終了時に自動的にTODOリストの進捗をチェックし、完了したタスクをアーカイブする

**トリガー**:
- ユーザーが「終了」「完了」「今日は以上」「セッションを終了」などと発言
- 大きなタスクブロックが完了した時
- ユーザーが明示的に「TODOをチェックして」と要求

**実行手順**:

```markdown
1. **進捗チェック**
   - `schedule_todo_list/` 内のアクティブTODOファイルをスキャン
   - 各ファイルの完了度（✅/⏳/❌）を確認
   - `schedule_todo_list/README.md` のダッシュボードと照合

2. **完了判定**
   - すべてのチェックボックスが `[x]` か確認
   - Acceptance Criteria がすべて満たされているか確認
   - 関連ビルド・テストが成功しているか確認

3. **アーカイブ判断**
   条件A（完了ファイル）:
     - ✅ すべてのタスクが完了
     - ✅ テスト・ビルド成功
     → `archive/` へ移動

   条件B（参考ドキュメント）:
     - 📋 より新しい版が存在する
     - 📋 レビュー・比較分析などの一時的ドキュメント
     → `archive/` へ移動

   条件C（進行中）:
     - ⏳ 1つ以上のタスクが未完了
     → そのまま `schedule_todo_list/` に残す

4. **ファイル移動**
   ```powershell
   # PowerShell コマンド例
   Move-Item -Path "schedule_todo_list\COMPLETED_FILE.md" `
             -Destination "schedule_todo_list\archive\"
   ```

5. **README更新**
   - `schedule_todo_list/README.md` のダッシュボードを更新
   - Active Files カウントを減算
   - Archived カウントを加算
   - 最終更新日を記載

6. **archive/README.md 更新**
   - アーカイブしたファイルをリストに追加
   - 完了日、内容、参照順序を記載

7. **完了レポート作成**
   ```markdown
   ## セッション完了サマリー

   **日時**: 2026-03-01 18:30

   **完了タスク**:
   - ✅ HONO_INTEGRATION_TODO.md (100%)
   - ✅ SECURITY_FIX_TODO.md (100%)

   **アーカイブ**:
   - 📦 archive/HONO_INTEGRATION_TODO.md
   - 📦 archive/HONO_REVIEW_CRITIQUE.md
   - 📦 archive/SECURITY_FIX_TODO.md

   **進行中** (次回セッション継続):
   - ⏳ 2026-03-01_BACKLOG.md (60%)
   - ⏳ PRODUCTION_DEPLOYMENT_TODO.md (40%)

   **次回優先タスク**:
   1. 🔴 Production Smoke Test
   2. 🔴 Environment Variables Verification
   ```

8. **ユーザーへの報告**
   - 簡潔な2-3文のサマリーを提示
   - 次回セッションの開始ポイントを明示
```

**自動化レベル**:
- 🟡 **半自動**: ユーザーのトリガーワード発言で実行
- 🔴 **完全自動**: 現時点では技術的に困難（Copilot Chat仕様上）

**エラーハンドリング**:
- ファイルが見つからない場合: README.md との不整合を報告
- 移動権限がない場合: PowerShell の `-WhatIf` で事前確認
- 依存関係がある場合: 相互参照をチェックしてユーザーに確認

## 🎯 ベストプラクティス

| 実践 | 理由 |
|------|------|
| ✅ **細かい単位でステップ分割** | 進捗追跡が容易、心理的疲労軽減 |
| ✅ **毎日EODで進捗更新** | 正確な完了度把握 |
| ✅ **完了条件を事前に明記** | ゴール曖昧化を防止 |
| ✅ **3～4日ごとにバックログ再生成** | 散在化防止、優先度見直し |
| ✅ **完了ファイルは即座にarchive** | スキーム参照の混乱防止 |
| ❌ **チェックボックスの手動変更のみ** | 自動化ツール検討 |
