# Draw.io Diagramming Skill

このスキルは、Draw.io MCP を使用して、プロジェクトのアーキテクチャやロジックを視覚化するためのガイドラインを提供します。

## 1. 基本コンセプト

- **Mermaid First**: 複雑な XML を直接生成するよりも、Mermaid 記法を生成して `open_drawio_mermaid` ツールで変換することを推奨します。
- **ドキュメントとの同期**: 重要な機能追加やリファクタリングの際は、`doc/architecture/` 等にダイアグラムを保存・更新してください。

## 2. 使用ツール

Draw.io MCP が提供する以下のツールを活用します：

- `open_drawio_mermaid`: Mermaid 記法から Draw.io ダイアグラムを生成します。
- `open_drawio_xml`: 既存の `.drawio` (XML) ファイルを編集する場合に使用します。
- `open_drawio_csv`: 構造化されたデータからダイアグラムを生成する場合に使用します。

## 3. デザイン・スタイルガイドライン

プロジェクトのデザインテーマに合わせた図解を心がけてください。

### テーマの適用
- **Glassmorphism (Light)**: 背景は薄いグラデーション、ノードは半透明の設定を意識した配色にします。
- **Elegant (Dark)**: 濃紺や黒を基調とし、アクセントカラー（青/紫）を使用して要素を強調します。

### 配置ルール
- **階層構造**: 上から下（Top-Bottom）または左から右（Left-Right）の流れを統一します。
- **コンポーネント分離**: フロントエンド、バックエンド、DB、外部サービス（Vertex AI等）を境界線で明確に区別します。

## 4. ワークフロー

1.  **要件分析**: 図解が必要な対象（システム全体、特定のフロー、データ構造）を決定します。
2.  **Mermaid 生成**: 対象のロジックを Mermaid 記法（graph, sequenceDiagram, erDiagram 等）で書き起こします。
3.  **Draw.io 展開**: `open_drawio_mermaid` を呼び出し、エディタを開きます。
4.  **保存**: 生成されたダイアグラムを `.drawio` 形式でプロジェクトの適切なディレクトリ（例: `doc/diagrams/`）に保存します。

## 5. 注意事項

- 図解は常に最新のコードベースを反映するようにしてください。
- 巨大すぎる図は避け、必要に応じてサブシステムごとに分割してください。
