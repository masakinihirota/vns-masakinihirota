# Writing Style Skill (Japanese)

このスキルは、日本語の技術ドキュメントや記事において、読みやすく一貫性のある文体を維持するための執筆ガイドラインです。

## 1. 文体・表現

- **文体の統一**: 「ですます調」を基本とし、「である調」との混在を避ける。
- **簡潔さ**: 一文は200文字以内を目安とし、長い文は分割する。
- **用語の選択**:
  - 口語表現（〜なんですが等）を避け、文語表現（〜ですが等）を使用する。
  - 曖昧な表現（〜と思う、〜のような）を避け、可能な限り断定する。
  - 「〜になります」「〜となります」を避け、変化がない場合は「〜です」とする。
- **助詞と送り仮名**:
  - 「〜たり」は必ず2回以上繰り返す（例：〜したり、〜したりします）。
  - 送り仮名の正確性を確認する。

## 2. 構成・論理

- **主述の一致**: 主語を省略せず、主語と述語のねじれがないか確認する。
- **能動態優先**: 受動態は極力避け、主語を明確にした能動態で記述する。
- **結論ファースト**: 結論→理由→詳細の順で構成する。
- **図示の活用**: 複雑な概念については文章だけでなく図示を検討する。

## 3. 表記ルール

- **ひらがな表記**: 形式名詞（こと、ため、ところ）、補助動詞（〜ください、〜いただく、〜できる）などはひらがなで表記する。
- **正式名称**: サービス名や製品名は公式サイトを確認し、大文字・小文字・スペースを含めて正確に表記する（例：Go, Spring Boot, Docker）。
- **略語**: 英語の頭文字をとった略語は大文字で表記する（例：API, CLI）。
- **記号**: 全角スラッシュは避け、半角スラッシュ「/」または「・」を使用する。

## 4. はてなブログ・Markdown固有

- **空行の挿入**: パラグラフ間や箇条書きの前後、コードブロックの前後には必ず空行を入れる。
- **太字**: `**太字**` の前後に半角スペースを入れない（はてなブログ仕様）。
- **リンク**: 「こちら」などの曖昧な言葉を使わず、リンク先の内容を明示したテキストとする。

## 5. チェックフロー

レビューや執筆時に、本スキルを参照して以下のチェックを行います：
1. 文体（ですます）が統一されているか？
2. 一文が長く、冗長になっていないか？
3. 形式名詞や補助動詞が適切にひらがな化されているか？
4. サービス名などの固有名詞が正確か？
5. 結論から述べられ、論理構成が明確か？
