# 統一用語定義 (Unified Terminology Standards)

このプロジェクトでは、ユーザー体験の一貫性を保つため、以下の用語の使用を厳格に統一します。コード内のコメント、UI上のテキスト、ドキュメント作成時に必ず参照してください。

## 1. 統一ルール一覧

| 旧用語               | 新用語 (統一後) | 備考                                                                         |
| :------------------- | :-------------- | :--------------------------------------------------------------------------- |
| **ルートアカウント** | **アカウント**  | シンプルさを優先。                                                           |
| **組織**             | **グループ**    | より親しみやすく一般的な表現を採用。                                         |
| **体験版**           | **お試し版**    | 「トライアル」や「体験版」が混在する場合は「お試し版」を検討（文脈による）。 |

## 2. 適用ガイドライン

- **UIテキスト**: 全てのページ、見出し、説明文、ボタンラベルに適用します。
- **コメント**: 技術的な説明やドキュメンテーション文字列も、可能な限り統一用語を使用します。
- **変数名/開発用語**: 既存のコードベースの互換性を考慮し、変数名（例: `organizationId`）の変更は慎重に行い、基本的には「表示・ラベル」の統一を優先します。

## 3. 確認フロー

- **新規実装時**: 実装計画（implementation_plan.md）作成時に、用語がこの基準に沿っているか確認する。
- **レビュー時**: PRレビューやAIによる自己監査時に、不一致がないかチェックする。
